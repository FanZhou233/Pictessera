# 图片内容标签与语义搜索 ToDoList

目标：让程序能够自动理解照片内容，为照片生成“苹果、桌子、水果、室内”等标签，并支持使用自然语言查找照片。全部分析默认在本地完成。

## P0 数据与基础架构

- [ ] 定义统一的 `ContentTag`：标签名、置信度、来源、模型版本、创建时间
- [x] 区分自动标签和用户标签，用户标签永不被重新分析覆盖
- [ ] 扩展 SQLite：保存照片标签、图片描述、模型版本和分析状态
- [ ] 增加标签的批量写入、删除、覆盖与版本迁移
- [ ] 为标签名称建立索引，支持中文、英文及别名检索
- [x] 使用文件稳定标识与内容签名复用结果，未变化照片不重复分析
- [ ] 模型升级后只将受影响记录标记为待重建
- [x] 模型分析失败安全回退，不影响普通扫描与浏览

## P0 自动内容标签

- [x] 建立可替换的 `ContentAnalysisProvider` 接口
- [x] 接入 Transformers 本地图像分类模型（仅加载用户指定目录，不自动下载）
- [x] 输出多个标签及置信度，例如：苹果 0.96、桌子 0.91、水果 0.88
- [x] 设定置信度阈值，避免低质量标签污染分类树
- [ ] 标签标准化：单复数、大小写、同义词和中英文映射
- [x] 支持一张照片拥有多个内容标签
- [ ] 在后台线程分析，限制并发和内存，不阻塞缩略图与浏览
- [ ] 为 HEIC/HEIF、JPEG、PNG、WEBP 提供统一解码入口
- [ ] LIVE Photo 默认分析代表静态帧，后续可选分析关键视频帧
- [x] 没有模型或模型加载失败时回退到现有轻量规则

## P0 标签搜索

- [x] 搜索框直接匹配自动标签和用户标签
- [x] 支持多标签交集：`苹果 桌子`
- [x] 支持任选匹配：`苹果 | 梨`
- [x] 支持排除条件：`水果 -人物`
- [x] 支持标签前缀：`标签:苹果`
- [ ] 搜索结果按匹配标签数和置信度排序
- [ ] 在搜索结果中显示命中的标签
- [x] 点击侧栏“内容识别”子项可浏览对应照片

## P1 语义搜索（以文搜图）

- [x] 提供本地 CLIP 图片/文字向量 Provider
- [x] 提供语义向量生成与本地缓存基础设施
- [x] 建立余弦相似度索引，避免每次搜索遍历全部照片
- [ ] 支持自然语言搜索：`木桌上的红苹果`
- [ ] 支持抽象搜索：`适合作为壁纸的海边照片`
- [ ] 支持中文查询；必要时在本地进行中英文查询归一化
- [ ] 标签匹配与向量相似度混合排序
- [ ] 搜索结果显示“语义相似度”，并允许调整严格程度
- [ ] 查询为空时完全绕过向量索引，不影响普通浏览性能

## P1 标签管理界面

- [ ] 图片详情页增加“标签”区域
- [ ] 标签使用可删除的胶囊样式展示
- [ ] 区分自动标签和手动标签的视觉样式
- [x] 允许通过右键“编辑标签”添加、删除和重命名用户标签
- [x] 支持多选照片后批量设置标签
- [ ] 自动标签旁显示置信度与模型来源
- [ ] 提供“隐藏此自动标签”和“始终保留”操作
- [ ] 提供单张照片“重新识别”按钮
- [ ] 空状态说明模型尚未安装、尚未分析或没有可靠标签

## P1 设置与任务管理

- [x] 设置 → 智能分类：内容识别总开关
- [ ] 设置识别强度：快速 / 平衡 / 精确
- [x] 设置最低标签置信度
- [ ] 设置分析设备：自动 / CPU / GPU
- [x] 设置并持久化本地模型路径
- [ ] 显示待分析、已完成、失败和跳过的照片数量
- [ ] 提供暂停、继续和取消后台分析
- [ ] 提供“重建内容标签”和“重建语义索引”按钮
- [ ] 提供模型缓存及向量缓存清理按钮
- [ ] 笔记本使用电池时可自动暂停高负载分析

## P2 Plus 与性能增强

- [ ] 基础版保留手动标签、轻量标签搜索和现有规则识别
- [ ] Plus 版启用真实视觉模型、自然语言搜索和向量索引
- [ ] 支持 GPU 加速，同时保证 CPU-only 环境可运行
- [ ] 大型图库采用分批提交与断点续作
- [ ] 对相似照片复用候选标签，再进行低成本校验
- [ ] 低内存设备自动降低输入分辨率和并发数
- [ ] 打包时模型采用可选下载，避免精简版安装包体积失控
- [ ] 模型文件校验完整性并显示许可证信息

## P2 隐私与可解释性

- [ ] 明确展示“照片、标签和向量均仅保存在本机”
- [x] 默认禁止任何云端识别请求，模型只从本地目录加载
- [ ] 若未来增加云端提供方，必须单独授权并明确上传内容
- [ ] 用户可导出、删除全部标签和语义向量
- [ ] 每个自动标签可以查看来源、置信度和生成时间

## 测试与验收

- [x] 单元测试：标签清洗、阈值、搜索表达式和向量排序
- [ ] 数据库测试：标签及向量写入、迁移、重载和删除
- [x] 增量测试：照片未变化时不重复推理
- [ ] 故障测试：模型缺失、损坏、内存不足和图片解码失败
- [ ] 性能测试：1 万张照片分类与搜索保持可交互
- [ ] UI 测试：单张编辑、多选批量标签、暂停和恢复任务
- [ ] 验收样例“红苹果放在木桌上”至少生成“苹果、桌子、水果”中的两个可靠标签
- [ ] 搜索“苹果”“桌上的水果”“不含人物的食物”均能返回合理结果
- [x] 关闭 Plus/AI 或模型失败后，普通扫描、缩略图、LIVE Photo 和基础分类不受影响

## 建议实施顺序

1. 数据模型、SQLite 与自动/手动标签边界。
2. 标签管理 UI 与精确标签搜索。
3. 本地内容识别 Provider 和后台任务队列。
4. CLIP 类向量、语义索引与自然语言搜索。
5. 性能优化、模型管理、隐私说明和 Plus 打包策略。
